Windows agent

The Windows agent will run on any 64bit Windows host such as Windows 7, 8, 10, Server 2008, 2012 and 2016. Simply extract the package on the host itself or an accessible network location. The Microsoft .NET 4.0 Framework is required. The agent can be installed by running the following command from an Administrator command line:

install.bat http://127.0.0.1/healthstone default

The first argument is the URL of this dashboard. The second argument is the template you want to use. Templates are stored in the templates folder on this dashboard server. The agent will fetch the template at every poll. The installer will copy the binary to %PROGRAMFILES%/healthstone, add the proper entries to the Registry under HKLM/Software/Healthstone and start the service.

Linux agent

The Linux agent will run on any modern Linux host. The only requirement is to have Python 3.x installed and available as python3 on the system. Simply extract the package on the host itself or an accessible network location. The agent can be installed by running the following command:

sudo ./install.sh http://127.0.0.1/healthstone default

The first argument is the URL of this dashboard. The second argument is the template you want to use. Templates are stored in the templates folder on this dashboard server. The agent will fetch the template at every poll. The installer will copy the Python script and add an entry to your rc.local startup file.

Templates configuration

Templates are flat files stored on the dashboard server which provides configuration information to agents. You can make as many templates as you want. Each template lists a number of modules that you wish the agents to process. Only the [General] section is mandatory, but each section kept must have all its keys listed. Do not use quotes or multiple lines for key values. Templates are not saved on hosts, each agent will fetch its specific template on every poll. Template filenames must contain only alphanumeric characters and end with .template. The same template can be used for Windows and Linux agents, although you may want to create multiple templates and divide them according to your network needs, such as a template for Windows servers, one for workstations, one for Linux, etc.
